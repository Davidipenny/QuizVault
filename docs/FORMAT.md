# 题库格式规范

本文档定义了 QuizVault 的题库数据格式，包括 Markdown 导入格式和 YAML 存储格式。

## 1. Markdown 导入格式

### 1.1 章节标题区分默认题型

在 Markdown 文件中使用 `##` 标题来设置后续题目的默认题型：

```markdown
## 判断题
（后续题目默认为 truefalse）

## 单选题
（后续题目默认为 single）

## 多选题
（后续题目默认为 multi）
```

标题匹配规则（宽松匹配，不区分大小写）：
- 标题文本中包含"判断"或"truefalse" -- 默认题型设为 `truefalse`
- 标题文本中包含"单选"或"single" -- 默认题型设为 `single`
- 标题文本中包含"多选"或"multiple" -- 默认题型设为 `multi`

### 1.2 单题类型覆盖

如果某道题的题型与当前章节默认类型不同，可在题目上方添加 HTML 注释覆盖：

```markdown
<!-- multi -->
**5. 这道题在单选题章节里，但实际是多选**
A. 选项A
B. 选项B
C. 选项C
D. 选项D

**答案：ABC**

**解析：** ...
```

支持的注释：
- `<!-- truefalse -->` -- 强制当前题目为判断
- `<!-- single -->` -- 强制当前题目为单选
- `<!-- multi -->` -- 强制当前题目为多选

注释仅影响紧跟其后的那一道题，之后的题目恢复使用章节默认类型。

### 1.3 题目格式

每道题由以下部分组成，各部分之间用空行分隔：

```markdown
**1. 题目文本（　）**
A. 选项A
B. 选项B
C. 选项C
D. 选项D

**答案：A**

**解析：** 解析内容

---
```

各元素说明：

| 元素 | 格式 | 说明 |
|------|------|------|
| 题号 | `**数字.**` | 必须是数字后跟句点，用 `**` 粗体包裹 |
| 题干 | `**题号. 题干文本**` | 包含在粗体标记内，通常以 `（　）` 结尾 |
| 选项 | `A. 文本` | 每行一个，以大写字母加句点开头，A-D 共 4 个 |
| 答案 | `**答案：X**` | 粗体包裹，冒号后为答案字母 |
| 解析 | `**解析：**` | 粗体标记"解析："后跟解析文本 |
| 分隔线 | `---` | 可选，用于分隔各题 |

### 1.4 完整示例

````markdown
## 判断题

**1. 实践是检验真理的唯一标准（　）**
A. 正确
B. 错误

**答案：A**

**解析：** 这是马克思主义认识论的基本观点。

---

## 单选题

**1. 毛泽东思想形成和发展的时代条件是（　）**
A. 俄国十月革命开辟了世界无产阶级社会主义革命的新时代
B. 1840年以后中国人民面临的反帝反封建任务
C. 五四运动开辟了世界无产阶级社会主义革命的新时代
D. 中国革命和建设的丰富实践

**答案：A**

**解析：** 毛泽东思想形成于20世纪前中期，当时的时代条件是俄国十月革命的胜利开辟了世界无产阶级社会主义革命的新时代。

---

<!-- multi -->
**2. 实际上是多选题（　）**
A. 马克思列宁主义在中国的运用和发展
B. 被实践证明了的关于中国革命和建设的正确的理论原则和经验总结
C. 中国共产党集体智慧的结晶
D. 毛泽东个人的思想

**答案：ABC**

**解析：** 毛泽东思想不等于毛泽东个人的思想，它是党和人民集体智慧的结晶。

---

## 多选题

**1. 毛泽东思想的活的灵魂包括（　）**
A. 实事求是
B. 群众路线
C. 独立自主
D. 与时俱进

**答案：ABC**

**解析：** 毛泽东思想的活的灵魂包括实事求是、群众路线、独立自主三个方面。

---
````

## 2. YAML 存储格式

题库数据存储在 `questions.yaml` 文件中（若未安装 PyYAML，自动 fallback 到 `questions.json`）。

### 2.1 文件结构

```yaml
meta:
  version: 1
  updated: "2026-06-28"
  source_files:
    - "毛概选择题（含解析）.md"
  total: 75
  truefalse_count: 10
  single_count: 40
  multi_count: 25

questions:
  - id: 1
    type: single
    question: "题目文本"
    options:
      - "A. 选项A"
      - "B. 选项B"
      - "C. 选项C"
      - "D. 选项D"
    answer: "A"
    explanation: "解析内容"
    source: "毛概选择题（含解析）.md"
    content_hash: "a1b2c3d4"
    flagged: false
    flag_reason: null
```

### 2.2 meta 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| version | int | 是 | 格式版本号，当前为 1 |
| updated | string | 是 | 最后更新日期，格式 YYYY-MM-DD |
| source_files | list | 是 | 导入来源文件名列表 |
| total | int | 是 | 题目总数 |
| truefalse_count | int | 是 | 判断题数量 |
| single_count | int | 是 | 单选题数量 |
| multi_count | int | 是 | 多选题数量 |

`meta` 中的 `total`、`truefalse_count`、`single_count`、`multi_count` 在每次保存时自动重新计算，无需手动维护。

### 2.3 questions 字段说明

每道题是一个列表项，包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | int | 是 | 题目编号，同类型内唯一（判断 1-N，单选 1-N，多选 1-N） |
| type | string | 是 | 题型：`"truefalse"`、`"single"` 或 `"multi"` |
| question | string | 是 | 题目文本（不含题号和括号） |
| options | list | 是 | 选项列表，判断题 2 个，单选/多选 4 个 |
| answer | string | 是 | 答案（见下方答案格式规则） |
| explanation | string | 否 | 解析内容，可为空字符串 |
| source | string | 否 | 来源文件名 |
| content_hash | string | 否 | 题目文本的内容哈希（MD5 前 8 位），用于去重 |
| flagged | bool | 否 | 是否标记为待修改，默认 `false` |
| flag_reason | string | 否 | 标记原因，无标记时为 `null` |

### 2.4 完整示例

```yaml
meta:
  version: 1
  updated: "2026-06-28"
  source_files:
    - "毛概选择题（含解析）.md"
```

## 3. 校验规则

导入工具 (`import_questions.py`) 在导入时对每道题执行以下校验。

### 3.1 必填字段

以下字段必须存在且不为空：

- `id` -- 题目编号
- `type` -- 题型
- `question` -- 题目文本
- `options` -- 选项列表
- `answer` -- 答案

缺少任一必填字段的题目将被拒绝导入。

### 3.2 题型校验

`type` 字段必须是以下三个值之一：

- `truefalse` -- 判断题
- `single` -- 单选题
- `multi` -- 多选题

### 3.3 选项校验

- 判断题：必须恰好 2 个选项（A、B）
- 单选/多选题：必须恰好 4 个选项（A-D）
- 选项以列表或字典形式存储均可，导入时自动转换

### 3.4 答案格式校验

**判断题（type: truefalse）：**
- 答案必须是 A 或 B
- 示例：`A`、`B`
- 错误示例：`C`（超出范围）、`AB`（多于一个字母）

**单选题（type: single）：**
- 答案必须是 A-D 中的恰好一个字母
- 示例：`A`、`B`、`C`、`D`
- 错误示例：`AB`（多于一个字母）、`E`（超出范围）

**多选题（type: multi）：**
- 答案必须是 2-4 个不重复的 A-D 字母
- 字母顺序无关紧要（`ABC` 和 `CBA` 等价）
- 示例：`AB`、`ABC`、`ABCD`
- 错误示例：`A`（仅一个字母）、`AAB`（有重复）、`ABCDE`（超出范围）

### 3.5 去重规则

- 基于题目文本 (`question` 字段) 计算 MD5 哈希值，取前 8 位作为 `content_hash`
- 计算前会去除题目文本首尾的空格和标点符号
- 同一哈希值的题目不会重复导入（除非使用 `--force` 参数）
- 单选题和多选题使用相同的题号范围（1-50 / 1-25），但 `content_hash` 前会加上类型前缀以区分

## 4. AI 生成题目模板

将以下提示词粘贴给 AI，可生成符合导入格式的题目：

````markdown
请按以下 Markdown 格式生成选择题：

## 判断题

**1. 题目文本（　）**
A. 正确
B. 错误

**答案：A**

**解析：** 解析内容

---

## 单选题

**1. 题目文本（　）**
A. 选项A
B. 选项B
C. 选项C
D. 选项D

**答案：A**

**解析：** 解析内容

---

## 多选题

**1. 题目文本（　）**
A. 选项A
B. 选项B
C. 选项C
D. 选项D

**答案：ABC**

**解析：** 解析内容

---

要求：
1. 判断题答案为 A 或 B，选项为"正确/错误"
2. 单选题答案为 A-D 中的一个字母
3. 多选题答案为 2-4 个不重复的 A-D 字母
4. 判断题必须有 2 个选项，单选/多选题必须有 4 个选项
5. 解析要说明正确答案的原因和其他选项的错误
6. 题号按顺序编号
7. 用 --- 分隔每道题
````
